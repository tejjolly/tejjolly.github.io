Project 1 · Research

Microvascular Resistance and Coronary Diagnostic Indices

BioSiMM Lab, Carnegie Mellon · 2024–present

Objective: Quantify how downstream microvascular resistance can shift FFR/CFR—and potentially change clinical decisions—independent of lesion geometry.

3D coronary CFD + LPN coupling HMR sweep (controlled) FFR/CFR computation Slurm/HPC campaigns
Poster (PDF)
Read IMRD write-up

Introduction: Social Value + Project Objective

Coronary artery disease assessment often relies on indices such as Fractional Flow Reserve (FFR) and Coronary Flow Reserve (CFR). In practice, patients can show discordant results that complicate treatment decisions. A key contributor is microvascular health: if the downstream microvasculature is abnormal or already maximally dilated, pressure- and flow-based indices may not behave as clinicians expect. This project isolates microvascular resistance as a variable and tests how it changes diagnostic readouts.

Methods

  • Built 3D coronary models and coupled outlet conditions to lumped-parameter networks representing distal beds.
  • Ran controlled sweeps of hyperemic microvascular resistance while holding upstream lesion geometry fixed.
  • Automated post-processing to compute FFR/CFR and related metrics consistently across cases.

Results / Evaluation

  • Produced comparative index tables across microvascular settings to identify sensitivity regimes (where indices are most/least stable).
  • Evaluated whether identical lesions can cross decision thresholds solely due to microvascular resistance changes (interpretation impact).

Discussion / Takeaways

The core takeaway is interpretability: FFR/CFR are not just “lesion severity meters.” They embed assumptions about distal physiology. The deliverable is a workflow that makes that dependence explicit, enabling more defensible, patient-specific interpretation.

My Role

  • Built/ran CFD campaigns, implemented LPN coupling assumptions, and automated analysis across parameter sweeps.

Skills emphasized: Coronary CFD · LPN modeling · ParaView/VTK automation · Slurm/HPC · Python analytics

Project 2 · Hemodynamics

Carotid Bifurcation Angle & Stenosis Influence on Hemodynamics

Carotid CFD Study · 2024

Objective: Determine how bifurcation angle and stenosis severity change low-/high-WSS exposure linked to plaque progression risk.

Angle: 50° vs 70° Stenosis: 0% vs 30% Pulsatile inflow WSS thresholds
Labeled video Report (PDF)
Read IMRD write-up

Introduction: Social Value + Project Objective

Carotid plaque burden and stroke risk correlate with hemodynamic environments, especially regions of persistently low wall shear stress (WSS). Because carotid anatomy varies widely across patients, it is useful to understand which geometric features systematically shift WSS distributions. This study tests how bifurcation angle and stenosis severity change WSS exposure in a controlled, comparative setup.

Methods

  • Generated a generalized 3D carotid geometry (CCA/ICA/ECA) and varied angle and ECA stenosis severity.
  • Prescribed a published pulsatile inflow waveform and tuned outlet boundary conditions to achieve realistic splits/pressures.
  • Computed surface area exposed to very low (< 5 dyn/cm²) and very high (> 70 dyn/cm²) WSS at key cycle phases.

Results / Evaluation

  • Quantified how stenosis increased low-WSS area at fixed angle; compared angle changes at fixed stenosis.
  • Used consistent thresholds and reporting so comparisons were not confounded by plotting choices.

Discussion / Takeaways

The key deliverable is a geometry-to-risk-factor mapping: even with identical boundary conditions, geometric changes can shift the spatial distribution of hemodynamic “risk markers.” This supports careful interpretation when translating population averages to patient-specific anatomy.

My Role

  • Built geometries/meshes, executed simulations, and implemented the WSS exposure analysis and summary reporting.

Skills emphasized: Vascular CFD · Meshing strategy · Pulsatile BCs · Post-processing & metrics · Technical reporting

Project 3 · Course Project

Decarbonization Building CFD: Reducing Re-Entrained Exhaust

Computational Fluid Dynamics · Fall 2024

Objective: Identify building shapes and intake/exhaust operating points that reduce re-entrainment and improve decarbonization efficiency.

External-flow CFD Passive scalar transport Parametric geometry study Re-entrainment metric
Scalar transport (GIF)
Read IMRD write-up

Introduction: Social Value + Project Objective

Direct-air-capture and related decarbonization facilities consume significant energy, so flow inefficiencies matter. Under adverse winds, cleaned exhaust can recirculate into the inlet, reducing CO₂ removal per unit energy. This project evaluates how geometry and flow settings influence re-entrainment so design decisions can be made early and defensibly.

Methods

  • Defined an atmospheric boundary-layer domain and candidate building geometries.
  • Simulated wake development and transported a passive scalar representing processed exhaust.
  • Compared cases using a consistent inlet “re-entrainment fraction” metric.

Results / Evaluation

  • Ranked geometries/operating points by re-entrainment risk under the tested wind conditions.
  • Identified design features that reduce wake recirculation and scalar return pathways.

Discussion / Takeaways

The value is not a single “best shape,” but a workflow: quantify re-entrainment as a measurable output and use it to guide early design choices that improve efficiency and robustness.

My Role

  • Set up transport metrics and contributed to case comparison and reporting; emphasized readable, decision-focused figures.

Skills emphasized: External-flow CFD · Species/scalar transport · Parametric studies · Communication of results

Project 4 · Computer Vision

PlayerVision: Real-Time Player & Ball Tracking for Sports

Computer Vision for Engineers · Fall 2024

Objective: Build a near real-time pipeline that identifies and tracks players and ball possession to reduce manual tagging overhead.

YOLO detection DeepSORT tracking Color-based team labeling Ball-carrier proximity
Read IMRD write-up

Introduction: Social Value + Project Objective

Broadcast analytics still relies heavily on manual annotation, which limits scale and consistency. A robust computer-vision pipeline can automate routine tagging tasks, enabling richer statistics and faster production workflows. This project prototypes a detection + tracking system for soccer broadcasts with interpretable, on-screen overlays.

Methods

  • Detected players and ball using a YOLO-based model; maintained identities over time using DeepSORT.
  • Implemented contrast enhancement and a pitch-masking approach to stabilize color signals.
  • Assigned team labels via jersey-color clustering; identified ball-carrier/nearest defender using proximity logic.

Results / Evaluation

  • Demonstrated continuous tracking with team labeling overlays on representative broadcast clips.
  • Validated stability qualitatively by checking identity continuity across camera motion and short occlusions.

Discussion / Takeaways

The main engineering lesson is robustness: practical systems need both strong models and “small” preprocessing choices that prevent failure under lighting changes, motion blur, and clutter.

My Role

  • Implemented preprocessing, jersey-color clustering, and possession/nearest-defender logic; integrated overlays for readability.

Skills emphasized: Python · OpenCV · Model integration · Tracking logic · Evaluation-minded prototyping
